# bebochka Frontend

React приложение для детского секонд-хенда.

## Технологии

- React 18
- Vite
- React Router
- Axios

## Установка и запуск

1. Установите зависимости:
```bash
npm install
```

2. Запустите dev сервер:
```bash
npm run dev
```

Приложение будет доступно по адресу: http://localhost:5173

## Настройка API

По умолчанию приложение использует API по адресу `http://89.104.67.36:55501/api`.

Для изменения URL API создайте файл `.env` в корне проекта:

```env
VITE_API_URL=http://89.104.67.36:44315
```

Или установите переменную окружения при сборке:

```bash
VITE_API_URL=http://89.104.67.36:44315 npm run build
```

## Сборка для продакшена

```bash
npm run build
```

Собранные файлы будут в директории `dist/`.

## Функциональность

### Публичная часть
- Просмотр каталога товаров
- Фильтрация и поиск (можно добавить в будущем)
- Отображение информации о товарах (название, бренд, описание, размер, цвет, цена, фотографии)

### Админка
- Управление товарами (CRUD операции)
- Добавление товаров с фотографиями
- Редактирование товаров
- Удаление товаров
- Выбор цвета из предустановленного списка

## Развертывание

### GitHub Secrets

Для автоматического развертывания через GitHub Actions добавьте следующие секреты в настройках репозитория:

- **SSH_HOST** - `89.104.67.36`
- **SSH_USERNAME** - имя пользователя для SSH
- **SSH_PRIVATE_KEY** - приватный SSH ключ
- **SSH_PORT** - порт SSH (обычно 22)
- **API_URL** - URL API бэкенда (опционально): `http://89.104.67.36:44315`

**Подробные инструкции:**
- `GITHUB_SECRETS.md` - детальное описание всех секретов с примерами
- `DEPLOYMENT.md` - инструкции по развертыванию

## Docker

### Сборка образа

```bash
docker build -t bebochka-frontend:latest -f Dockerfile .
```

### Запуск контейнера

```bash
docker run -d \
  --name bebochka-frontend \
  --restart unless-stopped \
  -p 80:80 \
  bebochka-frontend:latest
```

### Docker Compose

```bash
docker-compose up -d
```

## Настройка nginx

Файл `nginx.conf` настроен для проксирования API запросов на бэкенд сервер. Если API находится на другом адресе, обновите файл `nginx.conf`:

```nginx
location /api {
    proxy_pass http://YOUR_API_HOST:44315;
    # ... остальные настройки
}
```
